STEP 1: Push to GitHub (5 minutes)
────────────────────────────────────
  Open PowerShell in your project folder:

  $ cd d:\programs\food_donation\Food_Donation_Website
  $ git init
  $ git add .
  $ git commit -m "Initial commit: Food Donation Website"
  $ git remote add origin https://github.com/YOUR_USERNAME/food-donation-website.git
  $ git push -u origin main

  ✅ Code is now on GitHub!


STEP 2: Create Railway Account (3 minutes)
──────────────────────────────────────────
  1. Go to https://railway.app
  2. Click "Sign Up"
  3. Choose "Sign up with GitHub"
  4. Authorize and confirm

  ✅ Railway account created!


STEP 3: Deploy from GitHub (10 minutes)
───────────────────────────────────────
  1. In Railway dashboard, click "New Project"
  2. Click "Deploy from GitHub"
  3. Authorize Railway with GitHub
  4. Select "food-donation-website" repository
  5. Wait for build to complete (2-5 minutes)

  ✅ Your website is building!


STEP 4: Set Up Database & Variables (5 minutes)
───────────────────────────────────────────────
  1. Click "Add" (+) button
  2. Select "Database" → "MySQL"
  3. Wait for MySQL to provision
  
  Then add these variables to Backend service:
  
  PORT = 8080
  SPRING_PROFILES_ACTIVE = prod
  DB_URL = <paste from MySQL Connect tab>
  DB_USER = <username from MySQL>
  DB_PASSWORD = <password from MySQL>
  JWT_SECRET = foodDonationSuperSecretKeyForJwtTokenGeneration2026
  JWT_EXPIRATION_MS = 86400000

  ✅ Database connected!


STEP 5: Test Your Website (5 minutes)
─────────────────────────────────────
  1. Go to Railway Dashboard
  2. Click Backend service
  3. Copy the "Public URL" (e.g., https://food-donation.up.railway.app)
  4. Visit the URL in your browser
  5. You should see the Landing page!
  6. Try signing up and creating a donation

  ✅ Website is LIVE! 🎉

⚠️ BEFORE YOU START:
   • Ensure your code is committed and pushed to GitHub
   • Railway requires a GitHub repository
   • All configuration files are already created ✅

🔒 SECURITY:
   • Never commit .env files to GitHub
   • Use Railway's Variables feature for secrets
   • Change JWT_SECRET from default (done automatically in production)

📱 AFTER DEPLOYMENT:
   • Any code changes: git push → Railway auto-deploys
   • Update variables: Just change in Railway Dashboard
   • No downtime for environment variable changes

🆘 IF SOMETHING GOES WRONG:
   • Check Railway Logs tab for errors
   • Verify all environment variables are set
   • Ensure MySQL variables are correct
   • See DEPLOYMENT_GUIDE.md troubleshooting section
